# Repository Audit: tfplan2md Leftovers in GreenLedger

## Problem Description

GreenLedger was bootstrapped from or inspired by
[oocx/tfplan2md](https://github.com/oocx/tfplan2md), a .NET/C# Terraform-plan-to-Markdown
converter. Several configuration files, tool references, and a git submodule definition
still carry artifacts from the original project and do not belong in a Next.js carbon-
accounting application.

## Audit Methodology

1. Listed every file in the repository (`find . -not -path './.git/*'`).
2. Searched all files for keywords: `tfplan`, `terraform`, `plan2md`, `oocx`, `hcl`,
   `csharp`, `csproj`, `scriban`, `sbn`, `dotnet`, `nuget`, `azdo`.
3. Reviewed each hit to determine whether it is a leftover or a legitimate part of the
   GreenLedger Next.js project.
4. Inspected every root-level config file, workflow, agent definition, and skill for
   content that does not match a Next.js/TypeScript project.

## Recommended Cleanup Actions

### Priority 1 — Remove Entirely

1. **Delete `.gitmodules`** — The submodule definitions reference `oocx/tfplan2md-uat`
   (GitHub) and `dev.azure.com/oocx/test` (Azure DevOps). Neither submodule directory
   exists, and GreenLedger uses Docker-based UAT (see `.github/workflows/uat.yml`).

### Priority 2 — Remove Terraform MCP Tool from 5 Agent Definitions

Remove `'io.github.hashicorp/terraform-mcp-server/*'` from the `tools:` array in:

1. `.github/agents/code-reviewer.agent.md`
2. `.github/agents/developer.agent.md`
3. `.github/agents/issue-analyst.agent.md`
4. `.github/agents/requirements-engineer.agent.md`
5. `.github/agents/workflow-engineer.agent.md`

### Priority 3 — Fix Repository Owner in MCP Tool Examples

Replace `owner: "oocx"` with `owner: "StanislavMakhrov"` in:

1. `.github/skills/view-pr-github/SKILL.md` (8 occurrences)
2. `.github/gh-cli-instructions.md` (4 occurrences)

### Priority 4 — Clean Up .gitattributes

Remove .NET/C#-specific and Scriban-specific lines from `.gitattributes`:

```diff
- # C# files
- *.cs text eol=lf diff=csharp
- *.csproj text eol=lf
- *.slnx text eol=lf
- *.props text eol=lf
-
- # Scriban templates
- *.sbn text eol=lf
```

Also remove `.dll` and `.exe` from the binary files section (not needed for a
Next.js project):

```diff
- *.dll binary
- *.exe binary
```

## Impact Assessment

- **Risk**: Low — all changes are to configuration/documentation files, not application code.
- **Affected functionality**: Agent tool lists (removing an unused MCP server reference),
  MCP tool examples (fixing owner for correct repo targeting), git submodules (removing
  dead references), and gitattributes (removing rules for file types that don't exist).
- **Testing**: No code tests are affected. PR Validation should still pass since these are
  docs/config-only changes.

## Additional Context

- The repository remote is `https://github.com/StanislavMakhrov/green-ledger`
- The `src/` directory currently only contains `git.keep` — the Next.js app is not yet
  scaffolded, so there are no source-level leftovers to worry about
- The `oocx` GitHub user appears to be the original owner of tfplan2md from which this
  repo's tooling configuration was adapted
