# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |

## Reporting a Vulnerability

### Preferred: GitHub Security Advisory

1. Go to <https://github.com/StanislavMakhrov/green-ledger/security/advisories/new>
2. Fill in the vulnerability details
3. Submit the advisory

### Alternative: Email

Send an email to: **<support@green-ledger.dev>**

Include:

- Description of the vulnerability
- Steps to reproduce
- Impact assessment
- Any suggested fixes

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 5 business days
- **Fix Timeline**: Depends on severity

## Scope

### In Scope

- Application code vulnerabilities
- Dependency vulnerabilities
- Data exposure risks

### Out of Scope

- Issues requiring physical access to the host system
- Social engineering attacks
- Denial of service attacks against demo instances

## Security Best Practices

When using this application:

- ✅ Keep dependencies updated
- ✅ Review supplier form tokens for unauthorized access
- ✅ Use environment variables for sensitive configuration
- ✅ Run behind a reverse proxy in production

## Questions

- Open a [GitHub Discussion](https://github.com/StanislavMakhrov/green-ledger/discussions)

Thank you for helping keep this project and its users safe!
